# Architecture concepts

Krayin's architecture is built around two ideas: the **Repository Pattern** for clean data access, and a **Modular Design** that lets each feature live in its own package.

Start here:

- [Overview](./overview.md) — high-level tour of how Krayin is put together, the Repository Pattern, and the Modular Design.
- [Packages](./packages.md) — the full list of bundled Krayin packages and what each one does.
